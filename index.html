P2C Card Callebration Due date: 11/30/2024

Objective:

The objective of this project is to create an application that allows users to manually adjust P2C fields and FP files using an intuitive interface. This will enable our representatives to click and drag fields for alignment, reducing the need for manual editing and enhancing overall efficiency.

Steakholders:

Aaron Adams, Julian Lopez (tester), Keshav Dhawan, Shail Singh

Current Status:

This behavior is intentional and was demonstrated during the Show-N-Tell session. To clarify with an example:

In stage 3, the coordinates initially reflect the traced card. It is necessary to set the misplaced field coordinates to match their appearance on the printed output, aligning them with the physical card. When "Done" is clicked, the tool recalculates to account for these adjustments. For instance, if a field is meant to be at (10,10) but is displayed at (5,10) on the printed card, setting it according to its physical appearance will prompt the tool to adjust to (15,10), ensuring accurate placement when printed.

In summary, if a field is moved "n" pixels in any direction, the output will adjust "n" pixels in the opposite direction to maintain the expected position.

History:

It has been observed that when fields are adjusted, their placement appears to be inverted (moving fields down results in them being moved up). If there are any specific tests to be conducted, please provide the details!
